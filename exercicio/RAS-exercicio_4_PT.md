# Exercício 4 — Técnicas de Elicitação de Requisitos

## Objetivo

Este exercício tem como objetivo ajudar a compreender e aplicar o conceito de **elicitação de requisitos**, uma das atividades fundamentais do processo de Engenharia de Requisitos.  
Os alunos irão explorar como diferentes técnicas de elicitação podem ser utilizadas para recolher de forma eficaz os requisitos de clientes e utilizadores.

---

## Contexto

Nos exercícios anteriores, desenvolveu-se:

- **Exercício 1**: Um caso de uso (“Login de Utilizador”) utilizando o modelo fornecido.  
- **Exercício 2**: Uma lista de requisitos do sistema derivados do caso de uso.  
- **Exercício 3**: Uma versão refinada desses requisitos, reescrita de forma a eliminar ambiguidades.

Neste exercício, o foco será compreender **como esses requisitos poderiam ter sido elicitados** junto das partes interessadas durante as fases iniciais do projeto.

O sistema em análise continua a ser o **PICTURAS**, um serviço web online para edição de imagens.

---

## Descrição da Tarefa

O objetivo é analisar como os **requisitos do cliente e do utilizador** e os **casos de uso** relacionados com o PICTURAS poderiam ter sido elicitados na fase inicial do projeto.

A tua tarefa inclui:

1. **Identificar as partes interessadas (stakeholders)**
   - Quem são as pessoas ou grupos chave que devem estar envolvidos no processo de elicitação?  
   - Considera clientes, utilizadores finais, programadores, equipa de marketing, equipa de suporte, entre outros.

2. **Selecionar técnicas de elicitação adequadas**
   - Revê as técnicas de elicitação mais comuns descritas no material de apoio, tais como:
     - Entrevistas  
     - Questionários e inquéritos  
     - Workshops e sessões de brainstorming  
     - Observação direta ou estudos etnográficos  
     - Análise de documentos existentes  
     - Protótipos e demonstrações  
   - Escolhe pelo menos **três** técnicas e explica **por que razão** são adequadas ao projeto PICTURAS.

3. **Descrever como cada técnica seria aplicada**
   - Explica **que tipo de informação** procurarias obter com cada técnica.  
   - Fornece **exemplos de perguntas ou atividades** que poderias utilizar para recolher essa informação.

4. **Relacionar a informação elicitada com o Exercício 1**
   - Indica **que partes do caso de uso “Login” ou dos requisitos derivados** poderiam ter sido obtidas através de cada técnica.

---

## Resultado Esperado

O resultado deverá ser um pequeno relatório (1 a 2 páginas) com a seguinte estrutura:

### Título: Elicitação de Requisitos para o Projeto PICTURAS

1. **Identificação das Partes Interessadas**  
   - Lista das partes interessadas identificadas e respetiva relevância.

2. **Técnicas de Elicitação Selecionadas**  
   - Lista das técnicas escolhidas, com justificação para cada uma.

3. **Aplicação das Técnicas**  
   - Para cada técnica:  
     - Descrição de como seria aplicada.  
     - Exemplos de perguntas ou atividades.  
     - Ligação a requisitos ou casos de uso específicos.

4. **Reflexão**  
   - Indica quais os desafios que podem surgir durante a elicitação (por exemplo, barreiras de comunicação, necessidades contraditórias).  
   - Sugere estratégias para mitigar esses desafios.

---

## Orientações

Ao redigir o relatório:
- Utiliza uma linguagem clara e objetiva.  
- As justificações devem ser específicas ao contexto do PICTURAS.  
- Foca-te em **como obter informação de qualidade** junto das partes interessadas, que conduza a **requisitos claros e verificáveis**.  
- Lembra-te: a elicitação é a ponte entre *o que os utilizadores precisam* e *o que os engenheiros vão construir*.

---

## Resultados de Aprendizagem

Com a realização deste exercício, deverás ser capaz de:
- Compreender o propósito e a importância da elicitação de requisitos.  
- Saber selecionar e aplicar técnicas de elicitação adequadas.  
- Reconhecer o impacto da comunicação com stakeholders na qualidade dos requisitos.  
- Reforçar a ligação entre necessidades do utilizador, casos de uso e requisitos do sistema.
